1.1 算法

（1）计算

定义

可由一个给定计算模型机械地执行的规则或计算步骤序列称为该计算模型的一个计算

  • 一个计算机程序是一个计算
  • 计算可能永远不会停止
  • 不是算法

（2）算法

定义

  1. 算法是满足下列条件的计算：
  • 有穷性/终止性：有限步内必须停止
  • 确定性：每一步都是严格定义和确定的动作
  • 可行性：每一个动作都能够被精确地机械执行
  • 输入：有一个满足给定约束条件的输入
    • 函数可以没有输入
    • 算法必须有输入
  • 输出：满足给定约束条件地结果
  1. 算法的目的是求解问题
  2. 一个算法面向一个问题，而不是仅求解一个问题的一个或几个实例

算法的描述

  1. 伪代码

  2. 实例

    Input: A[1, ..., n] = n个数
Output: A[1, ..., n] = n个sorted数
FOR j=2 To n Do
    key <- A[j];
    i <- j-1
    WHILE i>0 AND A[i]>key Do
        A[i+1] <- A[i];
        i <- i-1;
    A[i+1] <- key;

（3）问题

定义

设Input和Output是两个集合。一个问题是一个关系R $\subseteq$ Input × Output

  • Input称为问题R的输入集合
    $$
    Input = { <a_1,….,a_n> | a_i是整数 }
    $$

    • Input的每个元素称为R的一个输入

  • Output称为问题R的输出或结果集合
    $$
    Output = { <b_1,….,b_n> | b_i是整数，且b_i \le…\le b_n}
    $$

    • Output的每个元素称为R的一个结果

  • 问题定义了输入和输出的关系 $\to$

$$
R = {
( < a_1, … ,a_n >, < b_1, …, b_n > ) | < a_1, … , a_n > \in Input, \
< b_1, … ,b_n > \in Output,
{a_1, … ,a_n} = {b_1, … ,b_n}
}
$$

1.3 算法分析引论

（1）算法的正确性

  • 一个算法是正确 $\to$ 对于每一个输入都最终停止，而且产生正确的输出

  • 不正确算法

    • 不停止
    • 对所有输入都停止，但对输入产生不正确结果

  • 近似算法

    • 对所有输入都停止
    • 产生近似正确的解或产生不多的不正确解

（2）算法正确性的证明

  • 证明算法对所有输入都停止
  • 证明对每个输入都产生正确结果
  • 调试程序 $\ne$ 程序正确性证明

（3）算法复杂性分析

  • 目的

    • 预测算法对不同输入所需资源量

  • 复杂性测度 $\to$ 输入大小的函数

    • 时间
    • 空间
    • I/O

  • 用途

    • 为求解一个问题选择最佳算法、最佳设备

  • 需要的数学基础

    • 离散数学
    • 组合数学
    • 概率论
    • 代数

  • 需要的数学能力

    • 建立算法复杂性的数学模型
    • 数学模型化简

（4）算法复杂性分析的度量

输入的大小（输入规模）

  • 设Input是问题R的输入集合
  • R的输入大小是一个函数F：Input $\to$ N, N是正整数集合
  • 输入规模量度
    • 最自然的量度：输入中的项数
    • 对于研究的每个问题将指出所使用的输入规模量度

时间复杂性

  • 一个算法对特定输入的时间复杂性是该算法对该输入产生结果需要的原子(基本)操作数或步数
  • 注意
    • 时间复杂性是输入大小的函数
    • 我们假设每一步的执行需要常数时间，实际上每步需要的时间量可能不同

空间复杂性

  • 一个算法对特定输入的空间复杂性是该算法对该输入产生结果所需要的存储空间大小

最坏复杂性

设Input是问题R的输入集合，$Complexity(X)$是求解R的算法A的复杂性函数，$size(y)$是确定R中输入大小的函数，A的最坏复杂性是
$$
Max{ Complexity,(size(y)),|,y\in Input }
$$
我们往往集中于只求最坏情况运行时间：

  • 最坏复杂性给出任何输入的运行时间的一个上界，能够确保该算法绝不需要更长的时间
  • 对某些算法，最坏情况经常出现
    • 例：在数据库中检索一条不存在的信息
  • 平均复杂性往往与最坏复杂性大致一样差
    • 例：随机选择n个数并应用插入排序

最小复杂性

$$
Min{ Complexity,(size(y)),|,y\in Input }
$$

平均复杂性

设 $y\in Input$, y作为算法A的输入出现的概率是$p_y$，A的平均复杂性为

$$
\sum_{y\in Input} p_y \times complexity(size(y))
$$

增长量级

  • 抽象：
    • 只考虑公式中最重要的项
    • 忽略低阶项
    • 忽略最重要项的常系数
  • 记作：**$\Theta(n^2)$**
  • 如果一个算法的最坏情况运行时间具有比另一个算法更低的增长量级，通常认为前者比后者有效

（5）算法分析的模型

随机访问模型(Random-Access-Model, RAM)

  • 单处理机
    • 串行执行
    • 无并发
  • 指令（所需时间为常量）
    • 算术指令
      • 加减乘除
      • 向上/向下取整
      • 取余
    • 数据移动指令
      • 装入
      • 存储
      • 复制
    • 控制指令
      • 条件/无条件转移
      • 子程序调用/返回
  • 基本数据类型
    • 整数型
    • 浮点实数型
  • 基本操作 $\to$ 每个操作常数的时间

并行多处理机模型(PRAM)

暂不涉及
